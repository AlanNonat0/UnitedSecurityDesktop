O United Security

A United Security surgiu em 2019, localizada em São Paulo com a premissa de implementar procedimentos de segurança aplicados a rotina e a forma que o cliente trabalha. Sempre buscando novos procedimentos de segurança para uma melhor interação de seus colaboradores para com os contratantes.

A empresa possui um conceito moderno de ferramentas e métodos como:

  • Planejamento estratégico
  • GGR - Google Rapid Response
  • Nmap
  • Snort

A United Security se compromete em seguir as diretrizes do "CIDAL" pois representam os principais atributos que são estabelecidos como base da Segurança da Informação instituídos pelos padrões internacionais ISO/IEC 27002:2013.

CIDAL:

  • Confidencialidade: propriedade que restringe o acesso a informação das entidades legítimas, ou seja, que são autorizadas pelo proprietário da informação;
  • Integridade: propriedade que garante que a informação manipulada mantenha as características originais que foram estabelecidas pelo proprietário da informação;
  • Disponibilidade: propriedade que garante que a informação esteja sempre disponível;
  • Autenticidade: propriedade que garante que a informação é vinda da fonte anunciada e que não foi manipulada ao longo de um processo de envio;
  • Legalidade: prioridade que institui que o uso da tecnologia de informática e comunicação sempre deve estar de acordo com as leis vigentes do local ou país;
  • Prestamos assistência na execução de procedimentos de segurança com praticidade, eficiência e transparência que se ligam ao nosso lema e como se aplicam nossos serviços.

Questões técnicas:

A United Security possui duas aplicações, uma WEB em php com integração com banco de dados, e outra desktop em C# também com integração ao banco.

Abaixo listamos as principais questões técnicas do sistema:

  • Linguagem versão Web: PHP versão:7.4.9, HTML, CSS, Javascript;
  • Banco de dados versão Web: MySQL versão: 10.4.14 - MariaDB;
  • Linguagem versão desktop: C# Visual Studio versão: 15.9.23
  • Banco de dados versão Desktop: SQL Server versão: 13.00.4001;
  • Autenticação de usuários: a autenticação de usuários pode ser feita tanto na própria base do sistema quanto através de uma base LDAP em algum ponto da rede.

Cadastro:

Abaixo listamos um passo a passo para realizar o cadastro de novos clientes e/ou novos funcionários:

  1. Ao fazer login na aplicação, clique em cadastro e será mostrado duas opções: Cadastro Cliente e Cadastro Funcionário;
  2. Clique na opção desejada e um formulário com as informações a serem preenchidas aparecerá ao lado;
  3. Preencha todas as informações pedidas no formulário, sendo que o código não é necessário por ser inserido sozinho em nosso banco de dados;
  4. No caso de cadastro de funcionário não há problema caso não tenha ainda uma matrícula, o campo não é obrigatório;
  5. No caso do cadastro de cliente não esqueça de selecionar qual(is) serviço(s) foram contratados pelo cliente em questão;
  6. Para consultar nossa base de dados, clique abaixo no botão "Consultar", no qual abrirá um formulário com a informação de todos os clientes e os serviços contratados, ou todos os funcionários;
  7. Caso seja necessário excluir algum registro, no formulário de consulta selecione o usuário desejado e clique no botão "Selecionar" que está na parte inferior do formulário;
  8. As informações do usuário selecionado aparecerão no formulário anterior. Então, basta clicar no botão "Excluir" e confirmar a interrogativa se tem certeza em excluir tal registro
